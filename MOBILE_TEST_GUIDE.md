# 移动端适配测试指南

## 快速开始

### 1. 启动开发服务器
```bash
cd stocker-front
npm run dev
```

服务器将在 `http://localhost:5173` 启动

### 2. 浏览器测试

#### Chrome/Edge 开发者工具
1. 打开浏览器访问 `http://localhost:5173`
2. 按 `F12` 打开开发者工具
3. 按 `Ctrl+Shift+M` (Windows) 或 `Cmd+Shift+M` (Mac) 切换到设备模拟模式
4. 在顶部工具栏选择设备:
   - **iPhone SE** (375x667) - 小屏手机
   - **iPhone 12 Pro** (390x844) - 标准手机
   - **iPhone 14 Pro Max** (430x932) - 大屏手机
   - **iPad** (768x1024) - 平板
   - **iPad Pro** (1024x1366) - 大平板

#### Firefox 响应式设计模式
1. 按 `Ctrl+Shift+M` (Windows) 或 `Cmd+Option+M` (Mac)
2. 选择预设设备或自定义尺寸

#### Safari (Mac)
1. 开发 -> 进入响应式设计模式
2. 选择 iOS 设备预设

## 常见问题排查

### 问题1: 页面显示不完整
**检查项**:
- 是否有固定定位元素遮挡内容
- 是否设置了正确的 padding-top (68px)
- 是否有元素设置了固定宽度

### 问题2: 横向滚动条出现
**检查项**:
- 是否有元素宽度超过100vw
- 是否有固定宽度的表格
- 是否有未适配的图片

### 问题3: 按钮点击不灵敏
**检查项**:
- 按钮尺寸是否足够大（最小44x44px）
- 是否有其他元素遮挡
- 是否设置了正确的 z-index

### 问题4: 输入框被键盘遮挡
**检查项**:
- 是否有固定底部元素
- 页面是否可滚动
- 是否需要调整视口设置

## 性能测试

### Chrome DevTools Performance
1. 打开 Performance 面板
2. 点击 Record
3. 滚动页面、点击按钮、切换页面
4. 停止录制
5. 查看性能指标:
   - FPS 应保持在 60
   - 无长时间的 JavaScript 执行
   - 无内存泄漏

### Lighthouse 测试
1. 打开 Lighthouse 面板
2. 选择 Mobile 设备
3. 运行测试
4. 查看评分:
   - Performance > 80
   - Accessibility > 90
   - Best Practices > 80
